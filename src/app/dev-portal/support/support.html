SLA Exclusions

The 99.9% Uptime SLA commitment excludes downtime resulting from the following scenarios:

  • Scheduled Maintenance: Maintenance windows communicated at least 7 days in advance.
  • Third-Party Outages: Outages caused by underlying telecom networks (e.g. Safaricom M-Pesa core downtime) or banking partners.
  • Force Majeure: Events outside our reasonable control (natural disasters, nationwide internet outages, regulatory freezes).
  • API Abuse: Downtime resulting from your breach of our rate limits or acceptable use policy.
  • Sandbox Environment: The Sandbox is provided "as-is" and is excluded from production SLA guarantees.

Uptime Calculation Methodology

Uptime is calculated per minute over the calendar month. A minute is considered "down" if more than 50% of API requests to the core payment endpoints return 5xx errors or time out.

Uptime % = (Total Minutes - Downtime Minutes) / Total Minutes * 100
  • Downtime is measured across all PayMo-managed core API nodes.
  • Planned maintenance (communicated 7 days in advance) is excluded.
  • Third-party failures (e.g., M-Pesa core down) are excluded.
